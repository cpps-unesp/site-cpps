Projetos guiados conectam estudo e colaboração. Eles devem ser pequenos o suficiente para orientar estudantes, mas próximos de problemas reais para gerar entregas úteis ao CPPS.

## Estrutura de um projeto guiado

Cada projeto deve ter:

- pergunta orientadora;
- fonte principal;
- pré-requisitos;
- passos reproduzíveis;
- dados de exemplo;
- entrega mínima;
- critérios de revisão;
- relação com projetos reais do CPPS;
- possibilidade de adaptação para Open edX.

## Entregas possíveis

| Tipo | Exemplo |
| --- | --- |
| Dataset | CSV, JSON ou banco pequeno com metadados |
| Script | Coleta, limpeza, validação ou conversão |
| Notebook | Análise exploratória com gráficos e comentários |
| Relatório | Texto curto com método, achados e limitações |
| Página da wiki | Guia que documenta o processo para novas turmas |
| Unidade Open edX | Conteúdo, exercício e avaliação derivados do projeto |
